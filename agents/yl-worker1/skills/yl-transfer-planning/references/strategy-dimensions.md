# 调拨策略维度与指标口径

> 步骤 1「缺口识别」见 `yl-supply-chain-analytics`；本文聚焦步骤 2「优先级 + 分配量」的多目标设计。

## 截图业务要点（步骤 2）

| 业务概念 | 系统口径 |
|----------|----------|
| **生产备货率** | `(现货 + 在途 + 本月已出库) / 本月销售计划` — 见 `yl_lateral_transfer.from_stock_rate_*` 注释 |
| **分配依据** | 按备货率文件 / 均衡规则排序，不是简单「缺多少补多少」 |
| **运输时效** | 仓间运输天数，影响到达后能否解未发 |
| **区域类型** | 核心仓 vs 偏远仓 — mock 可用 `WH_DOS_BASE` 高者视偏远 |
| **产出** | 补货优先级排序表、调拨分配比例表、分货计划草案 |

## 用户提出的四类主策略

### 1. 时效与服务率优先

**关注点**：未发订单、订单缺口、7 日需求窗口、在途能否及时衔接。

| 指标 | 计算 | 用途 |
|------|------|------|
| 服务紧急度 | `total_unship` + `GREATEST(0,-order_gap)` | 调入优先级 |
| 7 日需求 | `avg_plan_num * 7` | 是否 🔴 |
| 在途可解时间 | `yl_transit_inventory.remark` 预计天数 | 若 ≤3 天可「等的在途」少调 |
| 调拨后服务率 | `(现货+在途+建议量-unship)/plan` | 方案效果 |

**路径偏好**：同城/同区横向 > 近距基地正向 > 远距干线。

### 2. 成本最优化

**关注点**：仓网拓扑、运距、运费档、总调拨次数。

| 指标 | 计算 | 用途 |
|------|------|------|
| 运费档 | remark `运费档:低/中/高` | 路径排序 |
| 运距 | remark `NNNkm` | 同档内 tie-break |
| 调拨次数惩罚 | 方案内 `COUNT( DISTINCT 路径 )` | 少开单优先 |
| 单位成本 proxy | 档位的件·百公里单价 × 量 × 距离 | 方案总成本粗算 |

**路径偏好**：天津→济南（低/中）、武汉同城（低）；呼市→广州（高）尽量避免除非唯一解。

### 3. 新鲜度对冲

**关注点**：大日期去化、批次库龄、高流速仓洗货、基地新批次补给偏远。

| 指标 | 计算 | 用途 |
|------|------|------|
| 大日期占比 | `big_date_num / from_store_num_h` | 调出排序（高→低） |
| 库龄 P90 | `spot_inventory` 按仓 `age_days` | 批次风险 |
| 流速 | `sell_num / NULLIF(plan_num,0)` 或 `out_put_num` | 调入排序（高←低） |
| 消化月数 | `big_date_num / NULLIF(avg_plan_num*30,0)` | 不调则临期风险 |

**动作模式**：

- **横向**：大日期 + 低流速仓 → 高流速仓（冲量消化）
- **正向**：基地最新 `produce_date` 批次 → 偏远/长期缺货仓（空间换新鲜度）

### 4. 全网库存均衡（备货率趋同）

**关注点**：各调入仓调拨后备货率趋近同一目标，调出仓保留安全水位。

| 指标 | 计算 | 用途 |
|------|------|------|
| 当前备货率 | `(from_store_num_h + from_store_transit) / plan_num` | 偏离度 |
| 目标备货率 | `SUM(库存+在途) / SUM(plan_num)` 或业务给定 X% | 均衡锚点 |
| 安全水位 | `avg_plan_num * 14`（可调 7–21） | 调出下限 |
| 可转出量 | `(现货+在途) - 安全水位 - total_unship`，下限 0 | 调出上限 |
| 离散度 | 各仓备货率标准差 | 方案对比 |

**原则**：不把任一调入仓「喂饱到蓝牌」；只缩小仓间差异。

## 扩展策略维度（脑暴）

| 维度 | 关注点 | 关键指标 |
|------|--------|----------|
| **5. 最小干预** | 最少路径解决所有 🔴 | 调拨条数、总件数最小化 |
| **6. 订单履约优先** | 只解 `total_unship`，不管 DOS 美观 | `ship_gap<0` 的仓 |
| **7. 基地可分配约束** | 正向不超 `from_available` | 基地报表 + `yl_forward_transfer` |
| **8. 主供基地优先** | 销仓只从主供基地要货 | 呼市→华南、武汉→华中等网络 |
| **9. 渠道结构匹配** | 电商爆发仓多配 | `out_put_ec / out_put_num` |
| **10. 待检放行节奏** | 近期待检转合格作供给 | `from_store_num_d`、基地 `now_store_in` |
| **11. 推式纠偏** | 完成率低 + 高现货 → 减配 | `sell_completion_rate` 低且 DOS 高 |
| **12. 拉式紧急** | 完成率高 + 低现货 → 优先配 | 618 类场景 |
| **13. 双段联运** | 基地→枢纽→销仓 | 在途已有段 + 第二段横向 |
| **14. 风险分散** | 单仓调入不超过需求 120% | 避免二次压仓 |
| **15. 历史执行率** | 参考 `trans_num` vs `trans_num_jh` | 复盘调拨采纳率 |

## 多方案组合方式

| 用户意图 | 推荐组合 |
|----------|----------|
| 未指定 | 时效 + 均衡 + 新鲜度 [+ 成本] |
| 「快」 | 仅时效；对照均衡 |
| 「省」 | 成本 + 均衡 |
| 「清大日期」 | 新鲜度为主；时效兜底 🔴 |
| 「别动太多」 | 最小干预 + 均衡 |

## 方案互斥与约束（所有策略共用）

1. 调出后调出仓：备货率 ≥ 目标 − 5pp **且** DOS ≥ 7 天（可调）
2. 单次横向建议量 ≤ 调出 **可转出量**
3. 正向补货 ≤ 基地 `from_available`（有数据时）
4. 同一调入仓单方案累计 ≤ 缺口 × 1.2（防过补）
5. 待检库存不可当作可发量调出（仅合格现货参与）
